# **Project Plan**

### _Purpose_
The goal of this project is to create a researched based handbook for a 20 team startup. The goal of this handbook is to prevent unpredictabiity in project progress and eliminate defects in production. The handbook will define a standardised way of working across the team, helping new and existing team members follow consistent and proffesional software practices.

### _Deliverables_

A GitHub repository containing:
* Software quality handbook
* Three sections:
    * Task estimation in scrum
    * Code reviews
    * Automated Testing

* Each section includes: 
   * Overview of the topic
   * Key best practices
   * Common bad practices
    * 4-6 reference resources
    * Diagrams/images where relevant
* Retrospective 
* Contribution Table

### _Team Structure & Roles_

We aim to make this project as collaborative as possible so everyone will have a hand on three topics we touch on.

We will follow this structure for each topic:

2 people → Research + Writing

1 person → Review + Formatting

(roles will be rotated per topic)

Additionally:

1 person will define the initial formatting standard for all sections after review by the other two members.

### _Task Breakdown_

1. #### Project Setup:
    * Create GitHub repository

   * Define folder and file structure

   * Agree on communication tools (Teams & Snapchat)

   * Assign roles and topics

2. #### Research 
    * Find 4-6 high-quality resources

    * Focus on real-world experiences

    * Identify common themes and insights 

3. #### Writing 
    * Best practices
    * Bad practices
    * Examples/insights
    * Images where relevant

4. #### Review
    * Checks clarity and readability

    * Ensures formatting consistency

    * Suggests improvements via pull requests

5. #### Finalisation
    * Proofread entire handbook

    * Ensure consistency across sections

    * Add contribution table

    * Write retrospective

### _Communication_

We will have weekly meetings every friday via teams do discuss what needs to be done during the week/discussing and reviewing what has already been done. Our daily communication for quick updates will be via teams and snapchat.

### _Restrospective_
At the end of the project, the team will reflect on what worked well, the challenges we faced,how teamwork could be improved and what we would do differently next time. This will then be documented in the repository.
